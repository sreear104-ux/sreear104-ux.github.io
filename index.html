Title

Simple LangGraph Chatbot with Text File Memory

Objective

I created this chatbot to demonstrate a simple AI/ML application that can have a basic conversation with a user. The goal was to keep the project beginner-friendly while still showing how an AI assistant can be structured.

Process

I planned the chatbot using a simple design thinking approach. I identified the user need: a chatbot that could talk without requiring a complex website or database. Then I built a basic LangGraph flow using one chatbot node. I also added a temporary memory feature using a local text file so the bot could remember recent conversation history and delete it after a set time.

Title

Selecting the Right AI Approach: Predictive Maintenance and Speech Recognition

Project Overview

This course activity demonstrates my ability to compare traditional machine learning and deep learning approaches and determine which method is most appropriate for a real-world problem. The analysis focused on two applications: predictive maintenance in manufacturing and automatic speech recognition.

Objective

The objective was to select one traditional machine learning example and one deep learning example, explain why each approach was suitable, and evaluate why the alternative approach would be less effective.

Traditional ML Example

Application: Predicting equipment failure in a manufacturing facility. Algorithm: Random Forest. Random Forest is a strong fit because the data is structured and already organized into numerical features such as temperature, vibration, pressure, operating hours, and maintenance history. It can learn from labeled records and help identify conditions that may indicate an upcoming problem. This approach also uses fewer computing resources and offers more interpretability than deep learning.

Deep Learning Example

Application: Converting spoken conversations into written text. Algorithm: Transformer-based neural network. Speech recognition involves pronunciation, timing, accents, background noise, and language context. A transformer model can learn those relationships from large audio datasets and use surrounding words to interpret unclear speech. Traditional machine learning would depend too heavily on manually created audio and language features.

Skills Demonstrated
Machine learning Deep learning Algorithm selection Feature analysis Model comparison Technical communication
Key Takeaway

This activity showed that the most advanced model is not automatically the best choice. Model selection should depend on the structure of the data, available resources, required accuracy, interpretability, and the complexity of the problem.
